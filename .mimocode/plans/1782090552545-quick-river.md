# 项目熟悉报告

## 项目概述

这是一个**招聘求职平台**（类似Boss直聘）的微信小程序，支持两种用户角色：
- **个人用户**（求职者）
- **企业用户**（招聘方）

后端：Node.js/Express + MySQL + Redis，运行在 `http://localhost:3000`

---

## 项目结构

```
miniprogram-1/
├── app.js / app.json / app.wxss    # 小程序入口和全局配置
├── pages/          (30个页面)
├── components/     (14个组件)
├── custom-tab-bar/ (自定义TabBar，支持双角色切换)
├── services/       (领域服务层)
├── utils/          (工具函数)
├── styles/         (设计系统)
├── src/            (新架构：分层微内核)
├── node-jobs/      (后端服务)
└── ARCHITECTURE_RULES.md (架构治理规则)
```

## 前端服务层

### 三层架构（每个领域）
```
services/<domain>/
  ├── api.js     # HTTP请求
  ├── service.js # 业务逻辑
  └── model.js   # 数据模型
```

### 领域模块
- `core/` - 请求拦截、认证、Feed引擎、数据契约
- `job/` - 职位CRUD和Feed流
- `user/` - 用户资料
- `chat/` - 聊天消息
- `enterprise/` - 企业操作
- `ai/` - AI推荐
- `admin/` - 管理后台

## 新架构 (`src/`) - 分层微内核

依赖方向：`UI → modules → services → domain → core`

| 层 | 路径 | 职责 |
|----|------|------|
| Core | `src/core/` | 事件总线、插件管理、权限 |
| Domain | `src/domain/` | 纯数据模型、业务规则 |
| Store | `src/store/` | 响应式状态管理 |
| Services | `src/services/` | API调用+领域逻辑组合 |
| Modules | `src/modules/` | 业务编排，暴露接口给页面 |
| API | `src/api/` | HTTP请求+拦截器 |
| Plugins | `src/plugins/` | 扩展插件 |
| AI | `src/ai/` | 推荐管道 |
| Feed Engine | `src/feed-engine/` | 召回、过滤、排序 |

## 后端架构 (`node-jobs/`)

```
Routes → Controllers → Services → Models → MySQL
```

### 核心模块
- **认证**: JWT (7天过期), bcrypt密码加密
- **权限**: RBAC (super_admin/admin/enterprise/user) + 细粒度权限
- **多租户**: 所有查询按company_id隔离
- **AI**: DeepSeek API集成，混合评分 (0.7向量+0.3规则)
- **自动迁移**: 服务启动时自动建表/改表

### 主要API端点 (`/api/v1`)
- `/users` - 注册、登录、资料管理
- `/jobs` - 职位CRUD、搜索、AI搜索
- `/applications` - 投递管理
- `/chat` - 对话和消息
- `/ai/recommendations` - AI推荐
- `/enterprise/*` - 企业专属功能
- `/admin/*` - 管理后台

## 关键技术特点

1. **双角色系统**: 完全独立的页面集、TabBar和导航流
2. **Feed优先设计**: 多源合并(职位+AI+企业动态)、多因子排序、多样性平衡
3. **AI集成**: DeepSeek嵌入+规则评分的混合推荐
4. **动态页面**: 服务端驱动的UI渲染 (dynamic-page组件)
5. **轮询机制**: 未读消息10秒轮询，聊天4秒轮询
6. **零前端依赖**: 无npm依赖，纯微信原生API

## 问题检查结果 (共93个)

### 🔴 严重 (15个)
- home.wxml 4个事件处理函数未定义
- msg.wxml/JS 属性名不匹配
- detail.js 使用错误字段名导致聊天失败
- chat.js 无错误处理
- JWT密钥为占位符
- 企业接口无权限验证
- 上传接口越权
- 页面预览接口无认证
- 自定义TabBar硬编码localhost
- wx.switchTab用于非TabBar页面
- 缺少图片文件

### 🟠 高危 (26个)
- 认证状态不一致(两套请求系统)
- Badge索引错误
- 城市筛选失效
- SQL注入风险
- 错误信息泄露
- VIP状态过期
- 注册无审批

### 🟡 中等 (30个)
- 硬编码localhost URL
- 企业用户重定向问题
- 候选人统计去重错误
- CORS开放、无限流
- src/目录未排除打包

### 🟢 低危 (22个)
- 图标路径交换、孤立文件
- 重复方法定义
- 未使用组件注册

### 修复优先级
1. 立即: 后端权限验证、上传越权、JWT密钥
2. 尽快: 前端事件处理、属性匹配、错误处理
3. 计划: 统一请求系统、修复导航、清理硬编码
