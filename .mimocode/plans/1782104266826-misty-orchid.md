# AI 推荐功能完善计划

## 当前问题

| 问题 | 位置 | 说明 |
|------|------|------|
| loadMore 占位 | `aiRecommend.js:68-73` | setTimeout 500ms 后直接 `hasMore: false` |
| 筛选无效 | `aiRecommend.js:80-83` | 仅设置 UI 状态，无服务端筛选 |
| 感兴趣无动作 | `aiRecommend.js:101-103` | 仅 showToast，未投递 |
| 聊天未跳转 | `aiRecommend.js:95-104` | 无 chat 处理逻辑 |
| 不感兴趣无动作 | `aiRecommend.js:95-104` | 无 skip 处理逻辑 |
| 后端无分页 | `aiRecommendationRoutes.js` | 推荐接口一次性返回全部 |

## 修改方案

### 1. 后端：推荐接口支持分页 + 筛选

**文件**: `node-jobs/src/controllers/aiRecommendationController.js`

修改 `getUserRecommendations` 接口：
- 接收 `page`, `limit`, `filter` 参数
- `filter=all` 默认全部
- `filter=high` 只返回 `combined_score >= 80`
- `filter=new` 按 `created_at DESC`
- `filter=salary` 按 `salary_max DESC`
- 返回 `{ list, total, page, has_more }`

**文件**: `node-jobs/src/routes/aiRecommendationRoutes.js`

更新路由参数解析。

### 2. 前端：loadMore 接入后端分页

**文件**: `pages/aiRecommend/aiRecommend.js`

- `loadRecommendations()` 传递 `page=1, limit=10, filter` 给后端
- `loadMore()` 传递 `page+1` 并追加到列表
- 筛选切换时重新请求 `page=1`

### 3. 前端：卡片操作接入真实功能

**文件**: `pages/aiRecommend/aiRecommend.js` - `onCardAction`

| action | 逻辑 |
|--------|------|
| apply | 调用 `applications.apply({ job_id })` → 成功后移除卡片 |
| chat | 调用 `chat.createConversation(company_user_id)` → 跳转聊天页 |
| skip | 调用行为追踪 `behavior.track('skip_job', job_id)` → 移除卡片 |

### 4. 后端：跳过记录

**文件**: `node-jobs/src/controllers/aiRecommendationController.js`

新增 `skipRecommendation` 接口：
- `POST /api/v1/ai/recommendations/skip`
- 记录到 `user_action_logs`（action_type='skip_job'）
- 删除 `ai_job_recommendations` 中对应记录

**文件**: `utils/api.js`

新增 `ai.skipRecommendation(jobId)` 方法。
