# 前端需求文档

适用范围：招聘小程序前端、后台管理前端

目标：满足基础招聘业务使用，能完成登录、看职位、投递简历、收藏、沟通、企业审核和后台管理。

## 1. 角色

- 普通用户
- 企业用户
- 管理员

## 3. 核心功能

### 3.1 登录注册

- 支持手机号登录
- 支持普通用户注册
- 支持企业注册
- 登录后保存 `token`
- 请求接口时统一带 `Authorization: Bearer <token>`

### 3.2 职位

- 展示职位列表
- 支持关键词、地区、分类、薪资、经验、学历、工作类型筛选
- 支持职位详情查看
- 支持附近职位
- 支持职位新增、编辑、删除
- 支持职位状态展示：`draft`、`published`、`closed`、`archived`

### 3.3 企业

- 展示企业列表
- 展示企业详情
- 企业审核通过状态展示
- 支持企业审核、更新、删除

### 3.4 简历投递

- 用户可投递简历
- 用户可查看我的投递
- 企业可查看某职位收到的简历
- 企业可更新简历状态
- 简历状态建议展示：
  - `pending`
  - `viewed`
  - `interview`
  - `rejected`
  - `hired`

### 3.5 收藏

- 用户可收藏和取消收藏职位
- 用户可查看我的收藏

### 3.6 消息和会话

- 用户和企业可发消息
- 支持会话列表
- 支持会话详情
- 支持未读数展示
- 支持标记已读

### 3.7 通知

- 展示系统通知
- 支持标记已读
- 用于简历投递成功、企业审核通过、状态更新等提醒

### 3.8 后台管理

- 用户、企业、职位、简历、举报、消息、通知、审计日志都要能看
- 管理员可以审核企业
- 管理员可以查看审计日志

## 6. 简单交互要求

- 列表页要支持分页
- 详情页要支持返回上一页
- 表单提交前要做基础校验
- 接口请求失败要提示错误信息
- 未登录时跳转登录页
- token 失效时要清除本地登录状态
- 后台操作要显示加载中状态
